让 MCP 服务器检测到连接断开

```bash
# 7. 验证锁文件已清理
双击：测试锁文件检测.bat

预期输出：
✗ 锁文件不存在 → Houdini MCP 未运行 ✅
```

# 8. 验证可以再次启动

- 🎨 Houdini MCP 模式按钮应该是蓝色（可用）✅
- 不应该显示"已有实例运行"

### 预期结果

✅ **成功标准：**
- 关闭窗口后 2-3 秒，锁文件自动删除
- 可以再次启动 MCP 模式
- 不会提示"已有实例运行"

## 🐛 问题排查

### 如果窗口标题不正确

**可能原因：**
1. pywin32 未安装
2. 窗口出现太慢（超过15秒）
3. 进程 PID 匹配失败

**检查方法：**
```bash
# 1. 检查依赖
python -c "import win32gui, psutil; print('✓ 依赖正常')"

# 2. 查看启动器输出（如果有）
# 应该看到：✓ 窗口标题已修改：... → Claude-普通-0
```

**解决方案：**
```bash
双击：安装依赖.bat
```

---

### 如果锁文件不清理

**可能原因：**
1. MCP 服务器未检测到断开
2. 锁文件权限问题
3. 清理逻辑未执行

**检查方法：**
```bash
# 1. 查看锁文件路径
路径：C:\Users\<用户名>\.claude\plugins\houdini-mcp\.running.lock

# 2. 手动删除测试
删除该文件
再次启动 MCP → 应该可以启动
```

**临时解决方案：**
```bash
# 手动删除残留的锁文件
del "C:\Users\<用户名>\.claude\plugins\houdini-mcp\.running.lock"
```
