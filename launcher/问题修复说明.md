# 启动器问题修复说明

## 🐛 发现的问题

### 问题1：窗口标题设置错误

**现象：**
```
启动第1个普通模式 → 窗口标题是默认的
启动第2个普通模式 → 第2个窗口标题还是默认的
                   → 但第1个窗口标题变成了 "Claude-普通-1"
启动第3个普通模式 → 第3个窗口标题还是默认的
                   → 但第2个窗口标题变成了 "Claude-普通-2"
```

**根本原因：**
```python
# 错误的逻辑
def set_window_title(process_name: str, new_title: str):
    # 遍历所有 claude 进程
    for proc in psutil.process_iter():
        if 'claude' in proc.name.lower():
            # 找到第一个 claude 进程就修改
            # ❌ 问题：找到的是旧进程，不是新启动的进程！
            modify_window_title(proc.pid, new_title)
            break
```

**为什么会这样：**
1. 启动新的 Claude Code 进程（PID=1234）
2. `set_window_title` 在所有进程中搜索 "claude"
3. 它找到的是之前启动的旧进程（PID=1000）
4. 修改了旧进程的窗口标题
5. 新进程的窗口标题保持默认

---

### 问题2：锁文件清理失败

**现象：**
```
1. 启动 Houdini MCP 模式
2. 锁文件创建：~/.claude/plugins/houdini-mcp/.running.lock
3. 关闭 Claude Code 窗口
4. 锁文件仍然存在 ❌
5. 再次启动 MCP → 被拦截："已有实例运行"
```

**根本原因：**
```python
# 错误的假设
def main():
    lock_file.write_text("running")
    atexit.register(cleanup_lock)  # ❌ 只在进程退出时触发

    try:
        run_server()
    finally:
        cleanup_lock()  # ❌ 进程没退出，不会执行
```

**为什么会这样：**
1. Claude Code 是 Electron 应用
2. 关闭窗口 ≠ 退出进程
3. Claude Code 进程仍在后台运行
4. MCP 服务器进程也仍在运行
5. `atexit` 和 `finally` 都不会触发
6. 锁文件永远不会被删除

---

## ✅ 修复方案

### 修复1：使用进程 PID 精确定位

**新逻辑：**
```python
# 正确的逻辑
def launch_claude(use_mcp: bool, window_title: str = None):
    # 启动进程并获取 PID
    process = subprocess.Popen([claude_exe])
    new_pid = process.pid  # ✅ 获取新进程的 PID

    # 只修改这个新进程的窗口
    if window_title:
        set_window_title(new_pid, window_title)  # ✅ 传递 PID

def set_window_title(target_pid: int, new_title: str):
    # 只查找目标进程的窗口
    for hwnd in all_windows:
        window_pid = get_window_pid(hwnd)
        if window_pid == target_pid:  # ✅ 精确匹配
            set_title(hwnd, new_title)
            break
```

**工作流程：**
```
启动新进程 → 获取 PID (例如：5678)
    ↓
在后台等待窗口出现
    ↓
枚举所有窗口，查找 PID=5678 的窗口
    ↓
找到匹配的窗口 → 修改标题
    ↓
✅ 只修改新启动的窗口，不影响旧窗口
```

### 修复2：检测连接断开时清理锁文件

**新逻辑：**
```python
async def run_with_websocket(mcp, ws_server, lock_file):
    try:
        await mcp.run_stdio_async()  # 阻塞，直到 stdin 关闭
    finally:
        # ✅ stdin 关闭时（Claude Code 断开连接）清理锁文件
        if lock_file.exists():
            lock_file.unlink()
            logger.info("🧹 Cleaned up lock file (stdin closed)")
```

**工作原理：**
```
Claude Code 窗口打开
    ↓
MCP 服务器运行，stdin 保持打开
    ↓
用户关闭 Claude Code 窗口
    ↓
Claude Code 进程关闭与 MCP 的连接
    ↓
stdin 关闭 → mcp.run_stdio_async() 退出
    ↓
finally 块执行 → 清理锁文件 ✅
```

**多重保险：**
```python
# 1. stdin 关闭时清理（主要方式）
async def run_with_websocket(..., lock_file):
    try:
        await mcp.run_stdio_async()
    finally:
        cleanup_lock()  # ✅ 关闭窗口时触发

# 2. 进程正常退出时清理
atexit.register(cleanup_lock)  # ✅ 进程退出时触发

# 3. 异常退出时清理
try:
    run_server()
finally:
    cleanup_lock()  # ✅ 异常时触发
```
