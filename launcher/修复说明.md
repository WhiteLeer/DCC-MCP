# 🔧 MCP 配置问题修复说明

> **修复日期**: 2026-03-16
> **问题**: MCP 实例无法生效

---

## 🐛 问题描述

**症状**：
1. 使用启动器切换到 MCP 模式或普通模式后，配置没有生效
2. 即使"弃用"了 MCP 模式的 Claude，新启动的 Claude Code 仍显示 MCP 未运行
3. MCP 模式和普通模式在界面上看起来完全一样

**根本原因**：
- 启动器操作的是 **Claude Desktop** 的配置文件（`%APPDATA%\Claude\claude_desktop_config.json`）
- 但用户实际使用的是 **Claude Code**，它读取的是插件配置（`~/.claude/plugins/houdini-mcp/.claude-plugin/plugin.json`）
- 两个配置系统完全独立，互不影响！

---

## ✅ 修复方案

### 1. 删除 Claude Desktop 配置

```bash
# 删除整个 Claude Desktop 配置目录
rm -rf C:\Users\wepie\AppData\Roaming\Claude\
```

**原因**：
- 用户只使用 Claude Code
- Claude Desktop 配置不再需要
- 避免混淆

### 2. 修复启动器代码

**修改前**（错误）：
```python
# 操作 Claude Desktop 配置
CLAUDE_CONFIG_DIR = Path(os.getenv("APPDATA")) / "Claude"
ACTIVE_CONFIG = CLAUDE_CONFIG_DIR / "claude_desktop_config.json"
```

**修改后**（正确）：
```python
# 操作 Claude Code 插件配置
PLUGIN_DIR = Path.home() / ".claude" / "plugins" / "houdini-mcp"
PLUGIN_CONFIG = PLUGIN_DIR / ".claude-plugin" / "plugin.json"
```

**修改逻辑**：
```python
def enable_houdini_plugin(enable: bool):
    # 读取插件配置
    with open(PLUGIN_CONFIG, 'r', encoding='utf-8') as f:
        config = json.load(f)

    # 修改 enabled 字段
    config['enabled'] = enable

    # 保存
    with open(PLUGIN_CONFIG, 'w', encoding='utf-8') as f:
        json.dump(config, f, indent=2, ensure_ascii=False)
```

创建了 `测试配置.py` 和 `测试配置.bat`，用于验证配置是否正确。

## 🎯 工作原理

### 修复前（不工作）

```
启动器
  ↓
修改 Claude Desktop 配置
  ↓
Claude Code 启动
  ↓
读取插件配置（没有被修改）
  ↓
❌ 配置没有生效
```

### 修复后（正常工作）

```
启动器
  ↓
修改 plugin.json 的 enabled 字段
  ↓
Claude Code 启动
  ↓
读取插件配置（已被修改）
  ↓
  ├─ enabled=true  → 启动 MCP
  └─ enabled=false → 不启动 MCP
  ↓
✅ 配置正确生效
```
