===============================================
Claude Code 启动器 - 完整功能说明
===============================================

【核心功能】

1. 锁文件单实例检测
   - Houdini MCP 启动时创建锁文件：~/.claude/plugins/houdini-mcp/.running.lock
   - MCP 关闭时自动删除锁文件
   - 启动器检查锁文件判断是否已运行
   - 优势：不受端口自动顺延影响，检测准确可靠

2. 自定义窗口标题
   - 普通模式：Claude-普通-0、Claude-普通-1、Claude-普通-2...
   - Houdini MCP：Claude-Houdini MCP
   - 未来扩展：Claude-Maya MCP、Claude-Blender MCP...
   - 用途：任务栏中快速区分不同实例

3. 启动模式
   ┌──────────────────────────────────────────┐
   │ 普通模式（可多开）                       │
   │ - 窗口标题自动编号                       │
   │ - 无实例数量限制                         │
   │ - 每个窗口独立运行                       │
   │ - MCP 插件自动禁用                       │
   └──────────────────────────────────────────┘

   ┌──────────────────────────────────────────┐
   │ Houdini MCP 模式（单实例）               │
   │ - 窗口标题：Claude-Houdini MCP           │
   │ - 锁文件限制单实例                       │
   │ - 自动启用 Houdini MCP 插件              │
   │ - 启动后需打开 MCP Control GUI           │
   └──────────────────────────────────────────┘

===============================================
【安装步骤】
===============================================

1. 安装依赖
   双击：安装依赖.bat

   安装内容：
   - pywin32：用于修改窗口标题
   - psutil：用于进程检测

   注意：如果不安装 pywin32，窗口标题功能将被跳过，其他功能正常

2. 测试检测
   双击：测试锁文件检测.bat

   验证：
   - 未启动 MCP 时显示 "✗ 锁文件不存在"
   - 启动 MCP 后显示 "✓ 锁文件存在"

3. 启动使用
   双击：启动 Claude Code.bat

【技术细节】
===============================================

1. 锁文件机制
   路径：~/.claude/plugins/houdini-mcp/.running.lock

   内容示例：
   pid=12345
   started=1234567890.123

   生命周期：
   - MCP 启动时创建
   - MCP 正常退出时删除
   - MCP 异常崩溃时残留（需手动删除）

2. 窗口标题修改
   技术栈：
   - pywin32：win32gui、win32process
   - psutil：进程枚举

   流程：
   1. 启动 Claude Code 进程
   2. 后台线程等待窗口出现（最多10秒）
   3. 通过进程 PID 查找窗口句柄
   4. 使用 SetWindowText 修改标题

   注意：
   - 在后台线程中执行，不阻塞主程序
   - 如果没有 pywin32，功能自动跳过
   - 不影响 Claude Code 的其他功能

3. 实例编号
   存储：~/.claude/.normal_instance_counter

   逻辑：
   - 每次启动普通模式，读取并递增
   - 计数器持久化到文件
   - 不会重置（除非手动删除文件）

===============================================
【故障排除】
===============================================

问题1：启动器提示"MCP已运行"，但实际没有运行
原因：MCP 异常崩溃，锁文件残留
解决：手动删除锁文件
      路径：~/.claude/plugins/houdini-mcp/.running.lock

问题2：窗口标题没有修改成功
原因：未安装 pywin32 或窗口识别失败
解决：运行 "安装依赖.bat" 安装 pywin32
      注意：不影响其他功能，可忽略

问题3：普通模式编号重复
原因：计数器文件被删除
解决：正常现象，不影响使用
      或手动编辑：~/.claude/.normal_instance_counter

问题4：启动后找不到窗口
原因：Claude Code 路径配置错误
解决：检查启动脚本中的 CLAUDE_PATHS 配置
