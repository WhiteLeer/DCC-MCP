# Claude Code 启动器 - 更新日志

## v2.1 (2026-03-16) - 窗口标题修复 + 锁文件清理改进

### 🐛 已修复问题

#### 问题1：窗口标题设置错误
**现象：**
- 新启动的窗口标题是默认的
- 但之前的窗口标题变成了目标标题

**原因：**
- `set_window_title` 查找所有 claude 进程
- 找到的是旧进程而非新进程

**修复：**
```python
# 旧方案
subprocess.Popen([claude_exe])
set_window_title("claude.exe", title)  # ❌ 不知道是哪个进程

# 新方案
process = subprocess.Popen([claude_exe])
new_pid = process.pid
set_window_title(new_pid, title)  # ✅ 精确定位新进程
```

#### 问题2：锁文件清理失败
**现象：**
- 关闭 Claude Code 窗口后
- 锁文件仍然存在
- 无法再次启动 MCP

**原因：**
- Claude Code 关闭窗口后进程不退出
- `atexit` 清理不会触发

**修复：**
```python
async def run_with_websocket(mcp, ws_server, lock_file):
    try:
        await mcp.run_stdio_async()  # stdin 关闭时退出
    finally:
        cleanup_lock()  # ✅ 断开连接时清理
```

### 🔧 技术改进

1. **窗口标题设置**
   - 使用进程 PID 精确定位
   - 只修改新窗口，不影响旧窗口

2. **锁文件清理**
   - stdin 关闭时清理（主要）
   - 进程退出时清理（备份）
   - 异常退出时清理（保险）

## v2.0 (2026-03-16) - 锁文件 + 窗口标题

### 🎉 新增功能

#### 1. 锁文件单实例检测
**替代：** ~~端口检测~~ → **锁文件检测**

**问题：**
- ❌ 旧方案：检查端口 9876 是否占用
- ❌ 缺陷：MCP 有端口自动顺延机制（9876→9877→9878...）
- ❌ 结果：端口检测完全失效，可以无限启动 MCP

**解决：**
- ✅ 新方案：MCP 启动时创建锁文件
- ✅ 位置：`~/.claude/plugins/houdini-mcp/.running.lock`
- ✅ 优势：不受端口影响，检测准确可靠

**代码：**
```python
# 旧方案（已弃用）
def check_port_in_use(port: int) -> bool:
    # 检查端口 9876...
    pass  # ❌ 无法应对端口顺延

# 新方案
def is_houdini_mcp_running() -> bool:
    return HOUDINI_LOCK_FILE.exists()  # ✅ 简单可靠
```

---

#### 2. 自定义窗口标题
**目的：** 任务栏中快速区分不同 Claude Code 实例

**实现：**
- 普通模式：`Claude-普通-0`、`Claude-普通-1`、`Claude-普通-2`...
- Houdini MCP：`Claude-Houdini MCP`
- 未来扩展：`Claude-Maya MCP`、`Claude-Blender MCP`...

**技术栈：**
- `pywin32`：修改 Windows 窗口标题
- `psutil`：进程枚举和识别

**流程：**
```
启动 Claude Code
    ↓
后台线程等待窗口（最多10秒）
    ↓
通过 PID 查找窗口句柄
    ↓
SetWindowText 修改标题
```

---

#### 3. 普通模式自动编号
**存储：** `~/.claude/.normal_instance_counter`

**逻辑：**
```
第1次启动普通模式 → Claude-普通-0
第2次启动普通模式 → Claude-普通-1
第3次启动普通模式 → Claude-普通-2
...
```

计数器持久化，重启系统后继续递增。

```python
lock_file = Path.home() / ".claude" / "plugins" / "houdini-mcp" / ".running.lock"

def main():
    # 启动时创建锁文件
    lock_file.write_text(f"pid={os.getpid()}\nstarted={time.time()}")

    # 注册退出清理
    atexit.register(cleanup_lock)

    try:
        # 运行服务器...
    finally:
        cleanup_lock()  # 确保锁文件被删除
```
