## 🔒 单实例保护机制

### 自动检测

启动器会自动检测**端口9876**是否被占用：

- ✅ **未占用** → 可以启动MCP模式
- ⚠️ **已占用** → MCP模式按钮灰化，显示警告

### 冲突处理

**场景1: 尝试启动第二个MCP实例**
```
启动器自动检测到冲突
   ↓
MCP按钮灰化
   ↓
提示：已有MCP实例运行
   ↓
只能选择"普通模式"
```

**场景2: 已有普通模式实例运行**
```
无冲突（端口未占用）
   ↓
可以正常启动MCP模式
   ↓
两个实例互不影响
```

## ⚠️ 故障排查

### 问题1: "检测到已有MCP实例运行"但实际没有

**原因**: 旧进程未退出，端口被占用

**解决**:
1. 启动"Houdini MCP Control" GUI
2. 点击"🧹 清理旧进程"
3. 重新运行启动器

---

### 问题2: 启动器无法找到Claude Code

**错误信息**: "无法找到Claude Code可执行文件"

**解决**:
1. 检查Claude Code是否已安装
2. 查看启动器脚本中的路径配置：
   ```python
   CLAUDE_PATHS = [
       "%LOCALAPPDATA%/Programs/claude-code/Claude Code.exe",
       "%PROGRAMFILES%/Claude Code/Claude Code.exe",
       ...
   ]
   ```
3. 手动添加实际安装路径

---

### 问题3: Python未安装

**错误信息**: "Python未安装或脚本执行失败"

**解决**:
1. 安装Python 3.8+
2. 确保Python添加到系统PATH
3. 测试：`python --version`

## 🔧 配置文件说明

### 自动管理的配置文件

| 文件 | 用途 | 自动修改 |
|------|------|----------|
| `claude_desktop_config.json` | **活动配置** | ✅ 是 |
| `claude_desktop_config_nomcp.json` | 普通模式模板 | ❌ 否 |
| `claude_desktop_config_backup.json` | MCP模式备份 | ✅ 是 |

**位置**: `%APPDATA%/Claude/`

### 工作原理

```
启动MCP模式:
  活动配置 ← MCP配置（保持不变）

启动普通模式:
  1. 备份当前配置 → backup
  2. 复制nomcp配置 → 活动配置
  3. 启动Claude Code

切换回MCP:
  活动配置 ← backup恢复
```

## 🎓 技术细节

### 单实例检测原理

```python
def check_port_in_use(port: int) -> bool:
    """检查端口9876是否被占用"""
    # 尝试连接端口
    # 成功 → 端口被占用（MCP运行中）
    # 失败 → 端口空闲（可启动MCP）
```

### 配置切换机制

```python
# MCP模式：保持原配置
active_config = mcp_config

# 普通模式：临时替换
backup(active_config)
active_config = nomcp_config
```

### 进程隔离

- MCP模式：启动hython子进程，占用端口9876
- 普通模式：无子进程，无端口占用
- **结论**: 两种模式完全隔离，可并存
