## 🔧 常见操作

### 场景1: 每天首次启动

```
1. 双击"启动 Claude Code.lnk"
2. 选择"🎨 Houdini MCP 模式"
3. 双击"Houdini MCP Control"
4. 检查GUI显示：hython进程=1（绿色）
5. 开始工作！
```

---

### 场景2: 报错"handshake超时"

```
原因: 旧的hython进程未退出

解决:
1. 打开"Houdini MCP Control" GUI
2. 点击"🧹 清理旧进程"
3. 重启Claude Code
4. 完成！
```

---

### 场景3: 想要同时做其他工作

```
实例1（Houdini工作）:
  - 启动"MCP模式"
  - 打开GUI监控

实例2（其他工作）:
  - 再次双击启动器
  - 选择"普通模式"
  - 正常工作

✅ 两个实例互不干扰！
```

---

### 场景4: 修改了服务器代码

```
不需要重启Claude Code！

1. 打开GUI
2. 点击"🔄 重启 MCP 服务器"
3. 等待2秒
4. 代码修改已生效
```

## ⚠️ 故障排查速查表

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 启动器显示"插件未安装" | Claude Code未加载插件 | 重启Claude Code |
| "handshake超时" | hython进程堆积 | GUI → 清理旧进程 |
| GUI显示"无法连接" | MCP未启动 | 确认选择了"MCP模式" |
| 进程数显示>1（橙色） | 旧进程未清理 | 点击"清理旧进程" |
| 端口被占用 | 有MCP实例运行 | 关闭其他Claude Code窗口 |

## 💡 最佳实践

### ✅ 推荐做法
- 每天启动前先清理旧进程
- MCP模式专用于Houdini工作
- 其他工作使用普通模式（节省资源）
- 保持GUI开启以监控状态

### ❌ 避免做法
- 不要同时启动多个MCP实例
- 不要手动修改配置文件
- 不要在Houdini中手动运行Python脚本
- 不要手动删除plugin.json

---

## 🎓 工作原理

```
启动器 → 修改plugin.json的enabled字段
         ↓
Claude Code读取配置
         ↓
  enabled=true  → 启动MCP服务器
         ↓
  hython进程运行server_with_gui.py
         ↓
  ├─ MCP协议(stdio) → Claude Code通信
  └─ WebSocket:9876 → GUI控制面板
```
