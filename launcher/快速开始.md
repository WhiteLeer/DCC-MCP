# Claude Code 启动器 - 快速开始

## 🚀 功能特性

### ✅ 已实现
- **锁文件单实例检测**：Houdini MCP 单实例限制，不受端口顺延影响
- **自定义窗口标题**：任务栏中快速区分不同实例
  - 普通模式：`Claude-普通-0`、`Claude-普通-1`...
  - Houdini MCP：`Claude-Houdini MCP`
- **智能启动**：根据模式自动启用/禁用插件

### 🔮 未来扩展
- Maya MCP：`Claude-Maya MCP`
- Blender MCP：`Claude-Blender MCP`
- 3DMax MCP：`Claude-3DMax MCP`

## 📦 安装

### 1. 安装依赖
```bash
双击：安装依赖.bat
```

**依赖说明：**
- `pywin32`：修改窗口标题（可选）
- `psutil`：进程检测（可选）

> **注意：** 如果不安装依赖，窗口标题功能会被跳过，其他功能正常使用。

```bash
双击：测试锁文件检测.bat
```

验证锁文件机制是否正常工作。

## 🎯 使用方法

### 启动
```bash
双击：启动 Claude Code.bat
```

### 选择模式

#### 📝 普通模式（可多开）
- 窗口标题自动编号：`Claude-普通-0`、`Claude-普通-1`...
- 无实例限制
- MCP 插件自动禁用
- 适合日常开发、查文档、调试等

#### 🎨 Houdini MCP 模式（单实例）
- 窗口标题：`Claude-Houdini MCP`
- 单实例限制（锁文件检测）
- 自动启用 Houdini MCP 插件
- 启动后打开 MCP Control GUI

## 🔧 技术原理

### 锁文件机制
```
位置：~/.claude/plugins/houdini-mcp/.running.lock

流程：
1. MCP 启动 → 创建锁文件
2. 启动器检查 → 判断是否运行
3. MCP 关闭 → 删除锁文件
```

### 窗口标题修改
```
技术：pywin32 + psutil

流程：
1. 启动 Claude Code
2. 后台等待窗口出现
3. 查找进程对应的窗口
4. 修改窗口标题
```

## ❓ 常见问题

### Q1: 提示"MCP已运行"，但实际没有运行？
**A:** MCP 异常崩溃，锁文件残留。

**解决：** 手动删除锁文件
```
路径：~/.claude/plugins/houdini-mcp/.running.lock
```

### Q2: 窗口标题没有修改？
**A:** 未安装 pywin32 或窗口识别失败。

**解决：**
```bash
双击：安装依赖.bat
```
> 不影响其他功能，可忽略。

### Q3: 普通模式编号重复？
**A:** 计数器文件被删除。

**解决：** 正常现象，不影响使用。或手动重置：
```
路径：~/.claude/.normal_instance_counter
内容：0
```
