## ⚠️ 发现的问题及修复

### 问题1: 双重MCP配置 ⚠️ **【已修复，需清理】**

**问题描述**：
系统存在两套相同的MCP配置，会导致服务器启动两次。

**配置位置**：
1. **全局配置**: `~/.claude/mcp.json`
2. **插件配置**: `~/.claude/plugins/houdini-mcp/.mcp.json`

**当前状态**：
```json
// 两个文件内容完全相同（已统一）
{
  "mcpServers": {
    "houdini-mcp": {
      "command": "hython.exe",
      "args": ["-u", ".../server_with_gui.py"],
      ...
    }
  }
}
```

**影响**：
- ❌ 第一个实例占用端口9876
- ❌ 第二个实例自动切换到9877+
- ❌ GUI可能连接到错误的实例

**修复方案**：**删除全局配置，保留插件配置**

```bash
# 推荐方案：删除全局配置（让启动器控制）
del C:\Users\wepie\.claude\mcp.json

# 原因：
# - 启动器可以通过plugin.json的enabled字段控制MCP开关
# - 插件系统更灵活，支持单独管理
```

**已执行操作**：
- ✅ 统一了两个配置文件的格式
- ⏳ **待执行**：删除全局配置

---

### 问题2: pyproject.toml的entry point错误 ✅ **【已修复】**

**原配置**：
```toml
[project.scripts]
houdini-mcp = "houdini_mcp.server:main"  # ❌ server.py不存在
```

**修复后**：
```toml
[project.scripts]
houdini-mcp = "houdini_mcp.server_with_gui:main"  # ✅ 指向正确文件
```

**影响**：现在可以通过命令行运行 `houdini-mcp`

---

### 问题3: 缺少__main__.py ✅ **【已修复】**

**问题**：插件配置使用 `-m houdini_mcp.server_with_gui` 方式，但缺少入口文件。

**修复**：创建了 `houdini_mcp/__main__.py`

```python
from houdini_mcp.server_with_gui import main

if __name__ == "__main__":
    main()
```

**验证**：
```bash
hython -m houdini_mcp.server_with_gui --help  # ✅ 正常工作
```
